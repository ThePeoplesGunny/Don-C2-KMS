P1 Option B — d3.tree().nodeSize() Tidier Tree

RECOMMENDED

What you're seeing

d3.tree computes overlap-free positions using nodeSize([186, 146]). Parents center over children. Subtrees from different parents get 1.5x separation. No manual positions needed.

Architect

Virtual root handles multi-root views (don, acq).
Natural sizing — tree is its real size, zoom/pan to navigate. No squishing.
Manual overrides kept as delta offsets for fine-tuning, not absolute positions.

UI Compliance

Zero overlap — guaranteed by algorithm.
Subtree grouping — visual clusters match parent-child relationships.
Wide trees — 20+ leaves produce ~3700px width. Manageable with zoom but consider compact stagger for 6+ children.

Edge cases

Cross-cutting links (TA, DACO) become long arcs between subtrees. This is correct — it visually communicates "this authority crosses the hierarchy."

P2 Option A — Ghost Nodes for Dual-Hat Rendering

RECOMMENDED

What you're seeing

USFFC appears as a solid primary node in the ADCON tree (as COMLANTFLT). A dashed ghost appears in the OPCON tree (as NAVNORTH) with a gold dotted identity line connecting them. Same commander, different hat, different position in the hierarchy.

Architect

No new data fields — derived from existing dh + auth + dh_map.
d3.tree allocates space for ghosts as real nodes — no overlap possible.
Interactive — clicking ghost selects the same node. Side panel shows full data.

UI Compliance

Dashed border + 45% opacity = clearly "secondary reference," never confused with a real distinct node.
Gold identity line (straight, dotted) is visually distinct from all authority curves (Bezier).
Quad-hat nodes produce 3 ghosts — acceptable since they scatter across the tree, not clustered.

Creative addition

Gold halo rectangle around primary + ghosts (faint dashed outline) creates a visual group.

P2 Option B — Split Node + Badge Strip Alternative

ALTERNATIVE

What you're seeing

Top row: Split node — widened to 220px, divided into sections with authority-colored accent bars. Each section labeled with a hat designation.
Bottom row: Badge strip — standard 156px node with hat badges overlapping the top edge. Node stays uniform width.

Architect

Split node breaks d3.tree — variable width undermines P1's uniform nodeSize assumption.
Quad-hat at 280px — 76% wider than standard. Blows layout budget.
Badge strip preserves 156px width — works with d3.tree. Only adds 16px vertical.

UI Compliance

Split: 8px labels illegible at 40% zoom (renders as 3.2px).
Badge strip: readable, compact, authority-colored pills.
Badge strip complements P2-A — use badges on primary node + ghosts for spatial truth.

Recommendation

Split node: NO as primary solution.
Badge strip: YES as complement to P2-A ghost nodes.
